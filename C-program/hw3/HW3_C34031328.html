程式內容

distance 與 relation

先定義了兩個簡單的函數 distance(int N, int M) 和 relation(int N, int M) 。 distance 還蠻有用的，在其它的副函數中也會呼叫到。 可以減少巨集函數 SWAP(X,Y) 的使用； 使用 SWAP 很多時候是為了交換兩數的大小，避免相減後出現負數。 這時候就可以直接使用 distance 代替。

但有時候還是會用到 SWAP ，在單純判斷大小時。 而 relation 幾乎沒有用到；太麻煩了。 relation 用起來會像這樣： 

switch( relation(N,M) )
{
case '<' :
    printf("%d is littler than %d\n", N, M);
    break;
case '>' :
    printf("%d is greater than %d\n", N, M);
    break;
case '=' :
    printf("%d is equal to %d\n", N, M);
    break;
}

而我寫個 if 只要一行： 因為大部份的情況下，只考慮一種情況便足夠。 

if (N>M) printf("%d is littler than %d\n", N, M);

平方數數

這個問題還蠻複雜的；因為 C 語言不能算開根號……。 我的作法是從 1 開始，計算各數的平方。 如果在範圍內就次數加一。 （我在這裡唯一一次用到交換兩數的巨集……。）

數字的位數

digit 與 same 都是求和數字有關的問題，但我用了不同的解法。 因為 digit 是兩個數字各做一遍， 就算兩個都是 20 位數好了，也只有 40 遍。

而 same 我想到的解法是第一個數的第一個數字 和第二個數的所有數字比較，然後第二個數字，然後重複。 這樣太沒效率了，所以我改成一個個字判。 又一個個字判別時，用數學的方法處理有點麻煩； 所以把它拆成字串，再兩個數都跑一遍而己。

時鐘的角度

這題有趣了。我本來用浮點數作， 後來發現 浮點數不能取餘！ 為什麼要取餘呢？ 因為有時候相減後會出現負數或超過 360° ； 而取餘後應該會變正整數。

於是我把時鐘切成 12×60 度。 分針一分鐘動 12° ，時針一分鐘動 1° 。 於是就可以用整數寫了。 雖然後發現取餘後還是可能是負數。 改用 distance() 免掉負數的麻煩。

程式使用說明

輸入參數

此程式可以接受 命令列參數 。 （或說 main 函數被呼叫時可以接收參數。） 在 dos 下輸入完整 路徑/檔名 就可以執行該檔案。 像 HW3_C34031328.exe 的位置是 D:\NCKU\C_Program\HW3_C34031328.exe 。 打 D:\NCKU\C_Program\HW3_C34031328.exe 12 30 就可以呼叫該檔案，並傳入兩個參數 12 、 30 。 也就是程式內的 M 與 N 。 若沒有傳入參數，程式便會要求輸入。
